# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.1.x | Yes |

## Reporting a Vulnerability

Do not open a public issue for vulnerabilities that could compromise a server,
player data, economy data, inventories, permissions, or availability.

Use GitHub private vulnerability reporting if it is available for this
repository. If it is unavailable, contact the maintainer through the repository
owner profile.

Include:

- Affected plugin name and version.
- Server software and version.
- Java version.
- Other Luxorium plugins installed.
- Steps to reproduce.
- Impact assessment.
- Relevant logs or configuration snippets with secrets removed.

## Scope

Security-sensitive issues include permission bypasses, unsafe command execution,
data corruption, denial of service, unsafe Folia cross-thread access, and
exposure of private player or server data.
